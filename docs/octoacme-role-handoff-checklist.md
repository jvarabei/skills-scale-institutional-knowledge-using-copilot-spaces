# OctoAcme — Role Handoff Checklist

## Purpose
Ensure smooth transitions when team members join, leave, or change roles on a project. This checklist helps maintain continuity and knowledge transfer.

## When to use
- New team member joining the project
- Team member leaving or transitioning to another project
- Role change or responsibility shift within the project

---

## General Handoff (All Roles)

### Knowledge Transfer
- [ ] Schedule handoff session(s) with outgoing and incoming team members
- [ ] Share access to key repositories, tools, and communication channels
- [ ] Review project charter, roadmap, and current priorities
- [ ] Walk through active work items and their status
- [ ] Explain key decisions and their context
- [ ] Identify important historical context or tribal knowledge

### Documentation
- [ ] Update project roster with new team member information
- [ ] Add new member to relevant communication channels (Slack, email lists)
- [ ] Grant necessary permissions (GitHub, project boards, deployment tools)
- [ ] Share links to key documentation and resources
- [ ] Document any role-specific processes or preferences

### Introductions
- [ ] Introduce new member to stakeholders and key collaborators
- [ ] Schedule 1:1s with key team members
- [ ] Add to recurring meetings (standups, planning, reviews)

---

## Role-Specific Handoff Items

### Developer Handoff
- [ ] Walk through codebase architecture and key modules
- [ ] Share development environment setup instructions
- [ ] Explain build, test, and deployment processes
- [ ] Review coding standards and PR review practices
- [ ] Identify known technical debt and improvement opportunities
- [ ] Share access to development tools and environments

### Product Manager Handoff
- [ ] Review product vision and strategy
- [ ] Walk through backlog prioritization rationale
- [ ] Share customer research and user feedback sources
- [ ] Explain success metrics and how they're tracked
- [ ] Introduce to key stakeholders and their needs
- [ ] Review competitive landscape and market positioning

### Project Manager Handoff
- [ ] Review project timeline, milestones, and dependencies
- [ ] Walk through risk register and mitigation plans
- [ ] Explain status reporting cadence and format
- [ ] Share stakeholder communication preferences
- [ ] Review meeting schedules and facilitation notes
- [ ] Explain escalation paths and decision-making process

### Scrum Master Handoff
- [ ] Review team's sprint rhythm and ceremony format
- [ ] Explain team agreements and working norms
- [ ] Share team velocity data and capacity planning approach
- [ ] Identify known impediments and patterns
- [ ] Walk through retrospective action items and history
- [ ] Introduce team dynamics and collaboration style

### UX Designer Handoff
- [ ] Share user research findings and personas
- [ ] Review design system and component library
- [ ] Walk through design files and tools (Figma, etc.)
- [ ] Explain usability testing process and schedule
- [ ] Share accessibility standards and compliance needs
- [ ] Review feedback incorporation process

### Technical Writer Handoff
- [ ] Review documentation structure and information architecture
- [ ] Share style guide and terminology standards
- [ ] Walk through documentation tools and publishing process
- [ ] Explain release notes workflow
- [ ] Review feedback loops with Support and Product teams
- [ ] Share content calendar and priorities

---

## Handoff Completion

### Verify Readiness
- [ ] New team member can access all necessary tools and resources
- [ ] Key questions have been answered
- [ ] First tasks or responsibilities are clearly defined
- [ ] Support contacts are identified for questions
- [ ] First check-in scheduled for 1-2 weeks out

### Documentation Update
- [ ] Update team roster in project documentation
- [ ] Update RACI matrix or responsibility assignment
- [ ] Archive handoff notes in project documentation
- [ ] Update stakeholder communication list

### Follow-up
- [ ] Schedule 1-week check-in to address any gaps
- [ ] Outgoing member available for questions (if applicable)
- [ ] Scrum Master or PM monitors transition progress
- [ ] Address any access or knowledge gaps quickly

---

## Tips for Effective Handoffs
- Start handoff conversations early (2-4 weeks before transition)
- Use pair work sessions to transfer practical knowledge
- Record walkthroughs for future reference
- Create "day in the life" documentation for the role
- Be explicit about unwritten norms and expectations
- Don't assume prior knowledge—explain context
- Budget extra time for questions in the first sprint
